# Tablero de gestión en GitHub Projects

Fecha de configuración y consolidación: 15 de julio de 2026.

## Acceso

- [CrediBot - Gestión ágil](https://github.com/users/diegocalva04/projects/1)
- Repositorio de las historias: [JandrySan/CrediBot](https://github.com/JandrySan/CrediBot)

El proyecto es público y pertenece a `diegocalva04`, colaborador del repositorio. GitHub
no permite vincular un proyecto personal a un repositorio cuyo propietario sea otro
usuario; por eso las 20 tarjetas son issues reales de `JandrySan/CrediBot`, aunque el
contenedor del tablero esté bajo la cuenta del colaborador.

## Vistas y uso

- `Product Backlog`: vista tabular para revisar la trazabilidad y los campos ágiles.
- `Tablero Kanban`: vista operativa con columnas `Todo`, `In Progress` y `Done`.
- Estado al consolidar: 18 historias terminadas y 2 mejoras futuras en el backlog.

Cada historia contiene criterios de aceptación y está clasificada con:

- estado operativo de GitHub y `Estado ágil`;
- iteración (`Iteración 1`, `Iteración 2` o `Mejora continua`);
- prioridad MoSCoW;
- Story Points con escala Fibonacci;
- épica;
- responsable, etiquetas y milestone cuando corresponde.

## Integridad de la evidencia

El tablero se reconstruyó el 15 de julio de 2026 a partir del historial Git, los pull
requests, GitHub Actions y los documentos de gestión. No se modificaron fechas de
commits ni se afirmó que el tablero existiera antes de su creación. Los milestones de
las dos iteraciones representan los cortes cronológicos documentados y las historias
futuras permanecen abiertas para no presentar trabajo pendiente como terminado.

## Relación con la guía académica

El tablero materializa el Product Backlog priorizado, las estimaciones, las dos
iteraciones, la asignación de responsables y el estado de las historias. Las actas de
planning, revisión y retrospectiva se conservan como código en los documentos de cada
iteración para que también tengan revisión, historial y trazabilidad mediante Git.
